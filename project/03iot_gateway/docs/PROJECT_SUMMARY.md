## 🏗️ 系统架构详解

### 三层架构

```
┌─────────────────────────────────────────────┐
│          应用层 (Application Layer)          │
│  - 云平台数据上传 (TCP)                      │
│  - 业务逻辑处理                              │
│  - 数据格式转换                              │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│         核心层 (Core Layer)                  │
│  - poll事件循环                              │
│  - 线程池任务调度                            │
│  - 进程池管理                                │
│  - 令牌桶限流                                │
│  - 定时任务调度                              │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│         通信层 (Communication Layer)         │
│  - UDP接收 (STM32传感器)                     │
│  - TCP服务 (云平台连接)                      │
│  - IPC通信 (消息队列、共享内存、信号量)      │
└─────────────────────────────────────────────┘
```

### 数据流向

```
STM32传感器 
    │
    │ UDP (端口9527)
    │ 格式: "YYYY-MM-DD W HH:MM:SS DHT:T/H SHT:T/H ADJ:V"
    ▼
网关UDP接收
    │
    ├──> 令牌桶限流检查
    │
    ├──> 数据解析
    │
    ├──> 信号量P操作
    │
    ├──> 写入共享内存
    │
    ├──> 信号量V操作
    │
    └──> TCP上传到云平台
         │
         │ TCP (端口5924)
         │ 格式: JSON {"sensor_id":1,"temp":25.5,...}
         ▼
    云平台服务器
```

## 🔧 核心技术实现

### 1. Poll事件循环

```c
// 监听多个fd
poll_fds[0] = TCP监听套接字
poll_fds[1] = UDP套接字
poll_fds[2] = 信号管道
poll_fds[3+] = TCP客户端连接

while(running) {
    ret = poll(poll_fds, nfds, timeout);
    
    // 处理就绪事件
    if (TCP监听可读) -> accept新连接
    if (UDP可读) -> 接收传感器数据
    if (信号管道可读) -> 优雅退出
    if (客户端可读) -> 处理业务数据
}
```

### 2. 线程池任务队列

```c
// 任务队列结构
task_queue: [task1] -> [task2] -> [task3] -> NULL
                ↑                    ↑
             head                  tail

// 工作线程循环
while(1) {
    lock(mutex);
    while(queue_empty && !shutdown)
        wait(cond);
    task = dequeue();
    unlock(mutex);
    
    execute(task);
}
```

### 3. 令牌桶限流算法

```c
// 每秒补充令牌
alarm_handler() {
    token += cps;
    if (token > burst)
        token = burst;
}

// 消费令牌
fetch_token(n) {
    if (token >= n)
        token -= n, return n;
    else
        return 0;  // 限流
}
```

### 4. 共享内存+信号量同步

```c
// 写入数据
sem_p(semid, 0);        // 加锁
shm->sensors[i] = data;  // 写入
sem_v(semid, 0);        // 解锁

// 读取数据
sem_p(semid, 0);        // 加锁
data = shm->sensors[i]; // 读取
sem_v(semid, 0);        // 解锁
```

## 🔐 安全性考虑

1. **认证机制**：支持令牌认证，防止未授权访问
2. **PID文件锁**：防止多实例运行
3. **信号处理**：优雅退出，避免资源泄漏
4. **权限控制**：守护进程需要适当权限
5. **日志审计**：记录所有关键操作

## 🐛 已知问题

1. **需要root权限**：创建/var/run下的PID文件需要root权限
2. **IPC资源清理**：异常退出时可能残留IPC资源
3. **单点故障**：云平台连接断开后需要重连机制
