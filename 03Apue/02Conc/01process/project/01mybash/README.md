# LogHawk - 高性能分布式日志采集与处理系统

## 项目简介
企业服务器需7×24 小时持续运行，需要实时收集系统和应用日志。LogHawk 旨在解决传统日志收集方案中存在的采集延迟高、高并发下吞吐量不足、下游系统压力大且无流量保护、以及缺少可靠进程间通信机制等痛点。本项目通过模块化设计实现了三大核心模块的分离部署与独立扩展。

## 🎯 核心目标与特性
* **高性能采集**：系统能够实时监控日志文件。支持低延迟读取新增内容。提供断点续传功能。
* **高并发处理**：采用多进程架构并行解析日志。支持日志的过滤与格式化处理。
* **可靠缓冲机制**：利用共享内存加消息队列。实现无锁或低锁的高速数据传输。
* **精细化流量控制**：内置令牌桶算法以平滑输出流量。有效防止压垮下游存储系统。
* **稳定守护运行**：程序以守护进程模式运行。确保系统 7×24 小时不间断工作。
* **完备的进程管理**：支持单实例运行以避免多进程冲突。提供信号处理以支持平滑重启、优雅退出及异常捕获。

## 🏗️ 系统整体架构
LogHawk 由三大核心模块组成：

1.  **日志采集模块（Collector）**
    * 负责监控日志文件（如 `/var/log/app/*.log`）。
    * 读取新增内容并写入共享内存。
    * 记录读取偏移量以实现断点续传。
2.  **日志处理模块（Processor）**
    * 多进程并发消费共享内存中的数据。
    * 负责日志的解析、过滤与格式化（例如增加时间戳、提取关键字、JSON 封装）。
    * 将处理后的数据发送到消息队列。
3.  **日志输出模块（Outputer）**
    * 以守护进程形式运行并从消息队列消费日志。
    * 通过令牌桶限流保护下游。
    * 将日志写入本地文件或发送至远端系统。

模块间的通信关系如下：
* **Collector <-> Processor**：通过共享内存与信号量进行通信。
* **Processor <-> Outputer**：通过 XSI 消息队列进行数据传递。
* **内部控制**：依靠 `SIGALRM`、`SIGCHLD`、`SIGTERM` 等信号进行协同。

## 🛠️ 技术栈总览
本项目深度应用了 Linux 系统级编程技术：
* **文件 I/O**：使用 `open`、`read`、`lseek`、`stat`。
* **多进程管理**：使用 `fork`、`waitpid`、`exit`。
* **守护进程化**：使用 `daemon`、`setsid`、`umask`、`chdir`。
* **进程间通信 (IPC)**：涉及共享内存、消息队列以及信号量。
* **信号处理**：使用 `sigaction`、`setitimer`、`SIGALRM`。

## 🔒 企业级特性保障
* **稳定性**：守护进程后台运行。彻底脱离终端。
* **安全性**：支持单实例运行并利用文件锁防止多进程冲突。严格控制权限最小化。
* **可靠性**：信号量结合共享内存保证了进程同步。确保数据无丢失。
