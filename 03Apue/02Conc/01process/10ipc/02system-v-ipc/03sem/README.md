# 多进程并发控制与信号量实验

本实验通过四个阶段的代码演进，展示了在 Linux 环境下如何利用多进程处理计算密集型（质数查询）和 I/O 密集型（文件读取）任务，并重点演示了**信号量（Semaphore）**与**共享内存（Shared Memory）**的配合使用。

## 1. 实验文件说明

### A. 质数查询系列
* **`01prime_con.c` (静态分发模式)**
    * **机制**：4 个子进程通过 `100 + n` 的初始偏移量和 `+= 4` 的固定步长进行计算。
    * **特点**：任务在进程创建时就已“注定”，无需进程间通信，不存在竞争，但可能存在负载不均（某些区间质数多，某些少）。
* **`02prime_sem.c` (动态抢占模式)**
    * **机制**：利用**共享内存**存储当前的查询进度，4 个子进程通过**信号量互斥锁**竞争访问该进度。
    * **特点**：谁快谁多干。信号量确保了“取数 + 自增”操作的原子性，避免了多个进程处理同一个数字。

### B. 文件读取 (MyCat) 系列
* **`03mycat_nosem.c` (极端创建模式)**
    * **机制**：针对 100-300 之间的**每一个数字**都创建一个子进程进行判断。
    * **特点**：演示了不计后果创建进程的开销。父进程需要 `wait` 201 次，系统资源消耗极大。
* **`04mycat_sem.c` (池化抢占模式)**
    * **机制**：固定 4 个子进程（打工者）。共享内存记录文件偏移量 `offset`，信号量控制对 `offset` 的修改。
    * **特点**：每个进程抢到 4KB 的任务块后，**在临界区外**并行读取并打印。

---

## 2. 核心技术对比：静态分发 vs 动态抢占

| 维度 | 静态分发 (`01prime_con`) | 动态抢占 (`02/04_sem`) |
| :--- | :--- | :--- |
| **任务分配** | 提前根据 PID/编号 算好区间 | 运行时从共享池中实时抢夺 |
| **同步工具** | 无需工具 | **信号量 (PV操作) + 共享内存** |
| **负载均衡** | 较差 (容易出现“忙死”和“闲死”) | **极佳** (能者多劳，自动平衡) |
| **编程复杂度** | 简单 | 复杂 (需处理锁竞争和死锁) |

## 3. 信号量与共享内存的“配合公式”

在 `02prime_sem.c` 和 `04mycat_sem.c` 中，遵循了经典的并发控制模型：

1.  **P 操作 (sem_p)**：申请锁。如果信号量为 1，减为 0 顺利通过；如果为 0，进程挂起等待。
2.  **临界区 (Critical Section)**：
    * 从共享内存读取当前进度：`my_task = *shm_ptr;`
    * 更新共享内存进度：`(*shm_ptr)++;`
3.  **V 操作 (sem_v)**：释放锁。信号量加 1，唤醒其他等待进程。
4.  **并行区**：拿到任务号后，在**锁之外**执行耗时的计算或 I/O。

## 4. 实验重点笔记

* **原子性问题**：在 `04mycat_sem.c` 中，如果没有信号量，两个进程可能同时读到相同的 `offset`，导致文件块被重复读取。
* **SEM_UNDO 的作用**：在 `struct sembuf` 中设置 `SEM_UNDO`，可以确保某个子进程意外崩溃时，内核会自动释放它占用的信号量，防止整个系统死锁。
* **独立文件句柄**：在 `04mycat_sem.c` 中，每个进程在 `work` 中独立 `open` 文件。这是为了拥有独立的**文件偏移量指针**，配合 `lseek` 实现并行读取。
* **匿名 IPC**：实验中大量使用 `IPC_PRIVATE`，这适用于有亲缘关系的父子进程，避免了手动维护 `ftok` 密钥的麻烦。
