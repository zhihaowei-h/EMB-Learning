# 实验：Linux 进程替换与 exec 函数族学习

## 🎯 学习目标
1. 掌握 `exec` 函数族的核心功能（“灵魂替换”机制）。
2. 熟悉 `execl`、`execv`、`execvp` 等不同变体的使用场景及后缀命名规则（`l` 表示列表，`v` 表示数组，`p` 自动搜索 PATH）。
3. 深入理解进程替换前后的状态继承：明确被替换的进程会继承哪些属性（如 PID、文件描述符等“皮囊”），以及会被抹除哪些内容（代码段、数据段等“灵魂”）。
4. 理解可执行文件物理路径与进程身份（`argv[0]`）的解耦原理。

---

## 📂 代码实战与完成情况

### 1. 基础进程替换验证 (`01exec.c` & `02exec.c`)
* **功能描述**: 
  * `01exec.c`: 演示单进程直接调用 `execl`。直观验证了 `exec` 成功后原程序的代码流（如结尾的 `printf("End...\n")`）将被彻底截断丢弃。
  * `02exec.c`: 演示工业界标准的 `fork()` + `exec()` 范式。父进程创建子进程，并利用 `wait(NULL)` 等待子进程“变身”执行完毕。代码中还展示了数组传参（`execv/execvp`）与列表传参（`execl`）的等价替换方法。
* **完成情况**: ✅ 已掌握

### 2. 进程“马甲”与 argv[0] 欺骗演示 (`03exec_demo_caller.c` & `04exec_demo_target.c`)
* **功能描述**: 
  * 深入演示了 `execl` 函数的第二个参数（即传递给新程序的 `argv[0]`）的作用。
  * `caller` 作为发射器，向硬盘上的 `./target` 物理文件传递不同的虚假名字（如 `hello` 或 `bye`）。
  * `target` 程序通过检查自身的 `argv[0]` 表现出截然不同的行为分支。完美证明了同一个二进制文件可以根据调用者赋予的名字执行多态功能（类似 BusyBox 的底层原理）。
* **完成情况**: ✅ 已掌握

## 🛠️ 编译与运行指南

如果在 Linux 终端中测试这些代码，请使用以下命令：

```bash
# 1. 编译基础演示代码
gcc 01exec.c -o 01exec
gcc 02exec.c -o 02exec

# 2. 编译马甲演示代码（注意 target 的命名必须与 caller 内部调用一致）
gcc 03exec_demo_caller.c -o caller
gcc 04exec_demo_target.c -o target

# 3. 运行测试
./01exec
./02exec
./caller
