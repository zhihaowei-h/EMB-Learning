该行代码[01IOFBF.c](01IOFBF.c#L25)的执行原理/流程

#### 第一阶段：发起请求与阻塞
1.  **用户态：** **进程** 执行 `getchar()`，发现 **`stdin` 缓冲区**（用户态堆区）为空。
2.  **切换：** **进程** 发起 `read` 系统调用，**由用户态切换至内核态**。
3.  **内核动作：** **内核** 检查 **内核 TTY 缓冲区**，发现也为空。
4.  **挂起：** **内核** 将 **进程** 设为“睡眠状态”，将其挂在等待队列上，释放 CPU。**此时进程停在内核态内部，不再运行。**



#### 第二阶段：硬件触发与数据进入内核
1.  **外界动作：** 用户按下 `A` 和 `回车`。
2.  **执行者：** **终端驱动（内核层）**。
3.  **动作：** 驱动捕获硬件中断，将字符写入 **内核 TTY 缓冲区**。
4.  **唤醒：** **内核** 发现条件满足（收到了回车），将 **进程** 从等待队列中唤醒，设为“就绪态”。

#### 第三阶段：内核态内的“搬运”与返回
1.  **执行者：** **内核（`read` 系统调用的内核处理函数）**。
2.  **动作：** 进程在内核态继续执行，**内核** 将数据从 **内核 TTY 缓冲区** 物理拷贝到 **进程空间的 `stdin` 缓冲区**。
3.  **切换：** 拷贝完成，`read` 系统调用结束，**进程由内核态切换回用户态**。

#### 第四阶段：用户态最终读取
1.  **用户态：** **`getchar()`** 再次检查自己的 **`stdin` 缓冲区**，发现已经有货了（被内核填好了）。
2.  **动作：** **`getchar()`** 从中取走一个字符并返回。
